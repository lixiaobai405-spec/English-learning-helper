# 英语助手 (English Learning Helper) - PRD 与前端设计文档

## 一、 产品需求文档 (PRD)

### 1. 产品概述
“英语助手”是一款旨在提供极致简单、高效的中英文互译与语音交互工具。为了满足用户随时随地使用的需求，产品将采用 PWA (Progressive Web App) 技术，使得它在手机端和 PC 端均拥有媲美原生 App 的流畅体验。

### 2. 目标用户与使用场景
- **目标用户**：需要高频进行中英文翻译、发音练习、口语交流的用户。
- **使用场景**：
  - 手机端：走在路上单手操作，进行快速语音识别和翻译。
  - PC端：在工作或学习时，放置在屏幕一侧，进行长文本复制、粘贴与翻译。

### 3. 核心功能需求
1. **中英文双向互译**：自动或手动识别源语言并翻译为目标语言。
2. **语音识别 (STT)**：支持用户通过麦克风输入中文或英文，实时转换为文本。
3. **语音合成 (TTS)**：支持将翻译后的中文或英文结果通过高质量语音朗读出来。
4. **快捷操作**：支持一键清空输入框、一键复制翻译结果。

### 4. 非功能性需求
1. **PWA 支持**：可添加至桌面/主屏幕，支持离线缓存（核心外壳），无浏览器导航栏遮挡。
2. **响应式设计**：完美适配 Mobile (手机) 和 PC (电脑/平板) 两种屏幕形态。
3. **数据安全**：核心翻译与语音能力通过 Serverless 后端安全调用第三方高级 API（如 OpenAI / 百度等），确保 API Key 不在前端暴露。

## 二、 前端布局设计说明书

### 1. 设计原则
- **简洁高效**：消除所有不必要的视觉干扰，用户打开页面即可直接输入或说话。
- **功能聚合**：相关操作按钮与对应的文本区域紧密结合（卡片内嵌工具栏）。

### 2. 核心布局策略：响应式卡片 (Responsive Cards)

整个界面由两张核心卡片组成：**源语言卡片（输入）** 和 **目标语言卡片（输出）**。

#### A. 手机端布局 (Mobile View - 屏幕宽度 < 768px)
采用 **上下双卡片布局**，贴合手机竖屏握持习惯。
- **上方卡片（源语言/输入区）**：
  - 占屏幕约 40% 的高度。
  - 顶部：语言标识（如“中文”）。
  - 中部：大字号文本输入区。
  - 底部工具栏：左侧放置【🎤 语音输入】按钮，右侧放置【❌ 清空】按钮。
- **下方卡片（目标语言/输出区）**：
  - 占屏幕约 40% 的高度，背景色可做微弱区分（如浅灰），以示结果。
  - 顶部：语言标识（如“English”）。
  - 中部：翻译结果展示区。
  - 底部工具栏：左侧放置【🔊 朗读】按钮，右侧放置【📋 复制】按钮。

#### B. PC/平板端布局 (Desktop View - 屏幕宽度 ≥ 768px)
自动切换为 **左右双卡片布局**，充分利用宽屏空间。
- 屏幕水平分为两半。
- **左侧卡片（源语言/输入区）**：占据左侧 50% 宽度，内嵌工具栏布局与手机端一致，固定在卡片底部。
- **右侧卡片（目标语言/输出区）**：占据右侧 50% 宽度，背景色区分，内嵌工具栏布局与手机端一致。
- 这种布局使得用户在左侧打字时，视线可以自然平移到右侧查看结果，无需上下滚动。

### 3. 交互与状态设计
- **录音状态**：当用户点击【🎤 语音输入】时，按钮变为醒目的红色或动效状态，提示“正在聆听...”，再次点击停止录音。
- **朗读状态**：当点击【🔊 朗读】时，图标变为“播放中”动效，再次点击可停止朗读。
- **加载状态**：在请求翻译 API 时，目标语言卡片显示轻量的骨架屏 (Skeleton) 或“翻译中...”的过渡动画。
- **复制反馈**：点击复制后，按钮短暂变为“✅ 已复制”，随后恢复原状。

### 4. 技术选型建议
- **前端框架**：Next.js (React) 或 Nuxt.js (Vue) —— 推荐使用 Next.js，其自带的 API Routes 非常适合充当 Serverless 后端。
- **UI 框架**：TailwindCSS —— 极大地加速响应式布局（如 `flex-col md:flex-row`）和卡片样式的开发。
- **图标库**：Lucide Icons 或 Heroicons —— 简洁的线性图标，符合现代 App 审美。
- **PWA 配置**：使用 `next-pwa` 插件自动生成 Manifest 和 Service Worker。
- **部署方案**：Vercel（无缝支持 Next.js 和 Serverless API，自动关联 GitHub `main` 分支进行 CI/CD）。
